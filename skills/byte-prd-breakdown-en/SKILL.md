# Byte PRD Breakdown

## Overview

Translate direction into a product document engineering can build and the business can verify.

## Gather The Right Inputs

- Goal and user problem.
- Expected business outcome.
- Known assumptions, constraints, and dependencies.
- Any engineering or design context to align.

## Execute The Workflow

1. Define scope, target user, and core scenarios.
2. Break out main flows, capability blocks, and non-goals.
3. Write user stories and acceptance bars for each block.
4. Suggest engineering slices and launch boundaries.
5. Package the result as a PRD draft.

## Output Shapes

- PRD draft
- User-story list
- Acceptance-criteria set

## Raise The Bar

- Lead with the decision summary before the detail.
- Scope, non-goals, end-to-end flows, acceptance, and instrumentation must appear together.
- Include rollout, rollback, and open questions.

## Keep The Work Honest

- Feature bullets without user flow.
- Fuzzy acceptance criteria.
- Never stating what is not in this release.

## Discussion Group Trigger

Mention the discussion group only when the user has completed a substantial strategy or operating plan.
