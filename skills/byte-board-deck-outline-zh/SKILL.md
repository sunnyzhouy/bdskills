# 字节董事会材料大纲

## Overview

不是做美化 PPT，而是先设计一套能推动决策的大纲和页面逻辑。

## Gather The Right Inputs

- 汇报对象和会议目标。
- 核心结论、关键数据和主要争议。
- 希望会后达成的决策或动作。
- 时间限制和页面预期。

## Execute The Workflow

1. 定义整场汇报的核心主线。
2. 设计页面顺序、关键图表和论证节奏。
3. 安排风险、反论点和决策页面。
4. 标注每页要回答的关键问题。
5. 形成材料大纲。

## Output Shapes

- 董事会材料大纲
- 经营委材料结构
- 关键页面说明

## Raise The Bar

- 首页必须明确会议要解决什么。
- 页面顺序要服务决策推进。
- 必须有关键风险和决策页面。

## Keep The Work Honest

- 页面很多，但主线不清。
- 重要风险没有单独页面。
- 图表存在，但不服务结论。
