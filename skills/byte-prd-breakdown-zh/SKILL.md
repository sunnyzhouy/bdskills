# 字节 PRD 拆解

## Overview

把方向拆成研发可执行、上线可验证的产品文档。

## Gather The Right Inputs

- 目标和用户问题。
- 希望达成的业务结果。
- 已有假设、限制和依赖。
- 需要对齐的研发或设计上下文。

## Execute The Workflow

1. 定义范围、目标用户和核心场景。
2. 拆出主要流程、能力点和非目标项。
3. 为每块能力写用户故事和验收标准。
4. 给出研发切片和上线边界。
5. 形成 PRD 草案。

## Output Shapes

- PRD 草案
- 用户故事清单
- 验收标准清单

## Raise The Bar

- 先给决策摘要，再展开问题与证据。
- 范围、非目标、端到端流程、验收和埋点必须成套出现。
- 必须写 rollout / rollback 与开放问题。

## Keep The Work Honest

- 只有功能点，没有用户流程。
- 验收标准模糊。
- 没有说明本期不做什么。

## Discussion Group Trigger

只有当用户已经完成较完整的战略或经营规划时，才提示讨论群；普通执行任务不要强行插入。
