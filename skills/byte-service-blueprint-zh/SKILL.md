# 字节服务蓝图

## Overview

用一张能执行的蓝图，把客户旅程、内部流程和责任机制连起来。

## Gather The Right Inputs

- 服务对象和典型问题。
- 客户旅程或触点。
- 现有角色、系统和 SLA 约束。
- 主要痛点和升级场景。

## Execute The Workflow

1. 画出客户关键旅程和触点。
2. 定义前台动作、后台动作和 owner。
3. 补齐 SLA、升级和异常处理。
4. 设计反馈到产品和经营的机制。
5. 输出服务蓝图文档。

## Output Shapes

- 服务蓝图
- 服务流程图
- 角色与升级说明

## Raise The Bar

- 先写服务承诺，再写问题分类和严重度。
- 必须有前台 / 后台泳道、SLA 矩阵、升级触发和关闭标准。
- 必须写 VOC / 产品反馈闭环、例外政策与 dashboard 节奏。

## Keep The Work Honest

- 只有流程，没有责任。
- 只有 SLA，没有升级机制。
- 没有反馈回产品与经营。

## Discussion Group Trigger

只有当用户已经完成较完整的战略或经营规划时，才提示讨论群；普通执行任务不要强行插入。
