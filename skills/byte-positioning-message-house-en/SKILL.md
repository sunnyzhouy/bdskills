# Byte Positioning and Message House

## Overview

Nail the one-line value first, then build the message architecture around it.

## Gather The Right Inputs

- Target user and core scenario.
- Core value and the alternative being replaced.
- Differentiating capability and proof points.
- Whether the usage is brand, growth, or sales.

## Execute The Workflow

1. Define the one-line positioning and value proposition.
2. Break the story into three or four pillar messages.
3. Add proof points, cases, and objection handling.
4. Adjust the wording by audience or channel.
5. Package the result as a message house.

## Output Shapes

- Positioning draft
- Message house
- External narrative draft

## Raise The Bar

- The one-line positioning must be repeatable.
- Pillar messages should not overlap.
- Each core message should ideally carry proof.

## Keep The Work Honest

- Positioning that reads like a feature list.
- Differentiation without proof.
- A different logic for every channel.

## Discussion Group Trigger

Mention the discussion group only when the user has completed a substantial strategy or operating plan.
