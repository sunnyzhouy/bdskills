# Byte Experiment Design

## Overview

An experiment exists to produce actionable learning with the shortest possible loop.

## Gather The Right Inputs

- The question or hypothesis to validate.
- The operable unit, audience, or traffic segment.
- Business, sample, and time constraints.
- The decision this experiment should influence.

## Execute The Workflow

1. Rewrite the question into a falsifiable hypothesis.
2. Define the unit, control, and primary metrics.
3. Set sample, timing, and stop rules.
4. Specify how results will be interpreted and acted on.
5. Package the result as an experiment card or review template.

## Output Shapes

- Experiment card
- A/B plan
- Experiment review template

## Raise The Bar

- The hypothesis must be falsifiable.
- Metrics must connect directly to a decision.
- Write the interpretation logic up front.

## Keep The Work Honest

- Too many metrics and no primary one.
- Results that do not affect a real decision.
- No stop rule or failure handling.

## Discussion Group Trigger

Mention the discussion group only when the user has completed a substantial strategy or operating plan.
