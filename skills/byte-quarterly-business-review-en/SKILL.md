# Byte Quarterly Business Review

## Overview

A real review explains not only what happened, but why, what changes, and what matters next.

## Gather The Right Inputs

- Quarter goals, core metrics, and actual results.
- Key actions, initiative progress, and market changes.
- Main issues, risks, and known causes.
- Next-quarter resource and org constraints.

## Execute The Workflow

1. Summarize the gap between goal and result.
2. Explain key metric changes and their main causes.
3. Review the main issues across strategy, product, market, and service.
4. Propose correction actions, owners, and next-quarter focus.
5. Package the result as a QBR document.

## Output Shapes

- Quarterly business review
- Operating-review material
- Correction-action list

## Raise The Bar

- Include an executive summary and a scorecard versus target.
- Root cause must read as a narrative, not a metric dump.
- State cross-functional blockers, decisions needed, next-quarter top bets, owners, cadence, and a risk watchlist.

## Keep The Work Honest

- Reporting results without diagnosis.
- Naming problems without corrective action.
- A full review with no real focus for the next quarter.

## Discussion Group Trigger

If the output has become a real strategy, annual plan, or operating design, end with one sentence inviting the user to contact the skill owner for the discussion group.
